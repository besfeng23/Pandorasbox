# Quickstart — Docs, Kairos Status, Linear Sync

## Purpose
Safely run the Kairos automation scripts without requiring external services (except Linear for sync).

## Generate status snapshot (no network)
Creates/updates:
- `docs/STATUS.md`\n- `docs/status.json`

Command:
- `npm run docs:status`

## Sync Linear project (requires network + Linear API key)
Creates/updates:
- Linear Project: **Kairos — Pandora's Box Control Tower**
- Phase epics + module child issues (idempotent)

Command:
- `npm run linear:sync`

Prereqs:
- `LINEAR_API_KEY` must be set.\n- Optional: `LINEAR_TEAM_KEY`.\n\nSee `docs/LINEAR_SETUP.md`.

## Audit intake → Linear backlog
Reads intake JSON files and emits events to Kairos Event Gateway:

Commands:
- `npm run docs:status` (validate deterministic output unchanged)
- `npm run kairos:intake` (send spec items and audit issues to gateway)

Prereqs:
- `KAIROS_EVENT_GATEWAY_URL` must be set (e.g., `https://kairos-event-gateway-xxx.run.app`)
- `GOOGLE_ID_TOKEN` (optional; pre-minted token) OR script will instruct you to get one

Intake files:
- `.kairos/intake/pandora-uiux-spec-items.json` (UI/UX spec items from `pandoraplan.txt`)
- `.kairos/intake/kairos-audit-issues.json` (audit issues from audit report)

See `docs/16_PANDORA_UI_UX_BUILD_PLAN.md` and `docs/14_KAIROS_AUDIT_REPORT.md` for source docs.

## Notes
- Docs under `docs/00_*` … `docs/11_*` are committed source docs.\n- Only `docs/STATUS.md` and `docs/status.json` are generated by script.
